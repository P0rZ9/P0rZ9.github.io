分页知识详解

Posted on 2019-02-12 |

分页使用场景:当取到的数据量到达一定程度时,需要使用分页来进行数据分割

分页分类:

a.真分页(物理分页)
原理:每次点击下一页时去数据库查询。                
select * from user where id > 1 limit #{param1}, #{param2}    
优点:不会造成内存溢出    
缺点:翻页速度较慢
b.假分页(逻辑分页)
原理:一次将所有数据取出放入内存中,每次查询直在内存中取出数据。    优点:分页速度较快            
缺点:可能造成内存溢出

假分页容易实现,即准备一个集合保存从数据库中取出所有数据,根据当前码数,取出对应范围的数据,我们这里主要学习物理分页。

实现简单物理分页

1.准备一个Page工具类: 

public class Page {

  int start;
  int count;
  int total;

  //构造方法
  public Page(int start,int count){
      super();
      this.start = start;
      this.count = count;
  }

  //计算得到尾页
  public int getLast(){
      int last;
      //假设total=50  count=5
      if(total % count == 0)
          //最后一页开头为45
         last = total - count;
      else
          //不整除 最后一页开头为50
          last = total - total % count;

      last = last < 0 ? 0 : last;
      return last;
  }

  //是否有上一页 下一页
  public boolean isHasPreviouse(){
      if (start == 0)
          return false;
      return true;
  }
  public boolean isHasNext(){
      if(getLast() == 0)
          return false;
      return true;
  }

  //计算得到总页数
  public int getTotalPage(){
      int totalPage;

      if(total % count == 0)
          totalPage = total / count;
      else
          totalPage = total / count + 1;
      if(totalPage == 0)
          totalPage = 1;

      return totalPage;
  }

/* getter and setter */
}

存在的问题

a.没有边界判断,即在首页仍然可以点击前一页,不符合逻辑
b.会显示所有的分页,如果totalPage有100多页,则会全部显示出来,影响美观

改良版本

1.写好头与尾:

<nav class="pageDIV">
  <ul class="pagination">
  .....
  </ul>
</nav>

2.对« ‹增加边值判断 

<li <c:if test="${!page.hasPrevious}">class="disabled"</c:if>>
     <a href="?page.start=0">
         <span>
             «
         </span>
     </a>
 </li>

 <li <c:if test="${!page.hasPrevious}">class="disabled"</c:if>>
     <a href="?page.start=${page.start-page.count}">
         <span>
             ‹
         </span>
     </a>
 </li>

获取JavaBean的属性,只需要.符号就可以操作。${hero.name}就会自动调用其getName()方法。如果属性为boolean类型,就会自动调用isXXX方法。
所以使用page.hasPrevious就会自动调用isHasPrevious()方法。
并用js对其禁止:

<script>
  $(function () {
      $("ui.pagination li.disabled a").click(function () {
          return false;
      });
  });
</script>

3.中间页码的编写 

<c:forEach begin="0" end="${page.totalPage-1}" varStatus="status">
    <c:if test="${status.count*page.count-page.start<=30 && status.count*page.count-page.start>=-10}">
        <li <c:if test="${status.index*page.count==page.start}">class="disabled"</c:if>>
            <a href="?page.start=${status.index*page.count}"
               <c:if test="${status.index*page.count==page.start}">class="current"</c:if>
            >${status.count}</a>
        </li>
    </c:if>
</c:forEach>

从0循环到page.totalPage-1,varStatus相当于循环变量
status.count是从1开始遍历
status.index是从0开始遍历
要求:显示页码的前2个与后2个即可,即在第3页时,显示1,2,3,4,5的页码
测试条件:-10<当前页*每页显示数目-当前页开始编号<=30

4.后台实现分页
使用上面的Page工具类,我们在dao层定义一个List<User> list(int start, int count);方法,经过Mapper映射

<select id="list" resultMap="students">
    select * from user order by user_id desc limit #{param1}, #{param2}
</select>

然后在controller中:

// 获取分页参数
int start = 0;
int count = 10;

try {
    start = Integer.parseInt(request.getParameter("page.start"));
    count = Integer.parseInt(request.getParameter("page.count"));
} catch (Exception e) {
}
Page page = new Page(start, count);

List<User> users = userService.list(page.getStart(), page.getCount());
int total = userService.getTotal();
page.setTotal(total);
request.setAttribute("users", users);
request.setAttribute("page", page);

return "listUser";

即可完成分页功能

SSM项目中的分页

在SSM项目中,我们可以使用MyBatis的一款分页插件
PageHelper
使用方法:
1.pom.xml添加依赖包 

<!-- pageHelper -->
<dependency>
    <groupId>com.github.pagehelper</groupId>
    <artifactId>pagehelper</artifactId>
    <version>5.1.2-beta</version>
</dependency>

<!--jsqlparser-->
<dependency>
    <groupId>com.github.jsqlparser</groupId>
    <artifactId>jsqlparser</artifactId>
    <version>1.0</version>
</dependency>

2.spring-mybatis.xml配置文件

<!-- 配置SqlSessionFactory对象 -->
<bean>
......
    <property name="plugins">
        <array>
            <bean class="com.github.pagehelper.PageInterceptor">
                <property name="properties">
                    <!-- config params as the following -->
                    <value>
                        helperDialect=mysql
                    </value>
                </property>
            </bean>
        </array>
    </property>
</bean>

3.UserController:

PageHelper.offsetPage(page.getStart(),page.getCount());
List<User> users = userService.list();
int total = (int) new PageInfo<>(users).getTotal();
page.setTotal(total);
modelMap.addAttribute("page",page);
modelMap.addAttribute("users",users);

需要注意的是:PageHelper.offsetPage(page.getStart(),page.getCount);只对该语句以后的第一个查询语句得到的数据进行分页,所以这句话应该放在得到users前面。
